Библиотек для упрощения вызовов сервисов из клиентов.
Первоначально предназначалась для посторения взаимойдествия между J2ME мидлетом и сервером (2005 год)
поверх Сокета TCP или протокола HTTP

Описание интерфейса сервиса задается с помощью подмножества спецификации IDL.
Построитель Skeleton и Proxy сервиса в язык java реализован  в модуле Coder.
Есть так же построитель для С# (первоначально предназначалось для PocketPC проект Coder_net) он 
совместим с построителем java, т.е. можно сервис например на java а клиенские вызовы делать на C#

Сервис может быть реализован как на стороне сервера и вызываться клиентом, 
так и на стороне клиента и вызваться сервером. В последнем случае сценарий взаимодействия 
выглядит так 
	1. Клиент размещает у себя сервис для вызова, регистрируя реализацию (имплементацию) построенного по idl Skeleton
	в транспортном слое
	2. Клиент иницирует соедиенение с сервером.
	3. После  установления соединения сервер связывает канал (транспортного слоя) взаимодействия с клиентом с построенным по idl Proxy
	4. После этого на стороне сервера можно осуществлять вызовы бизнес логики клиента через интерфейс описанный на idl
